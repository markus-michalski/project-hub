# Stakeholders — Event Project

> **Template file.** Copy to `~/.project-hub/knowledge/event/stakeholders.md`
> and replace placeholders with your event's actual team.

## Purpose

Defines everyone involved in the event — organiser team, vendors, speakers/performers —
and the communication plan for keeping them informed.

## Communication Plan

Who receives which communications and when.

| Audience | Communication | Owner | Channel | When |
|----------|---------------|-------|---------|------|
| All attendees | Registration confirmation | Automated | Email | Immediately on registration |
| All attendees | Event logistics (venue, schedule) | Registration Lead | Email | D-14 |
| All attendees | Reminder | Registration Lead | Email | D-3 |
| All attendees | Day-before reminder + contacts | Registration Lead | Email | D-1 |
| All attendees | Thank you + survey | Registration Lead | Email | D+1 |
| All attendees | Follow-up resources / recording | Event Manager | Email | D+7 |
| Speakers | Speaker briefing | Event Manager | Email | D-28 |
| Speakers | Slide deadline reminder | Event Manager | Email | D-14 |
| Speakers | Day-of logistics + contact | Speaker Liaison | Email | D-1 |
| Vendors | Briefing documents | Venue Coordinator | Email | Per vendor timeline |
| Vendors | Final headcounts / specs | Venue Coordinator | Email | D-3 to D-5 |
| Vendors | Payment confirmation | Event Manager | Email | Per contract terms |
| Internal team | Runsheet distributed | Event Manager | Email / Slack | D-3 |
| Internal team | On-site crew briefing | Event Manager | In-person | D-1 or morning of event |
| Sponsor / Stakeholder | Save the date | Event Manager | Email | D-60 |
| Sponsor / Stakeholder | Programme update | Event Manager | Email | D-14 |
| Sponsor / Stakeholder | Post-event summary | Event Manager | Email | D+7 |
