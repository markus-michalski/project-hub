# Merchant Onboarding Project

## Key Fields

- **market**: Target country/market (DE, NL, SE, NO, FI, AT, CH, ...)
- **products**: Payment products being integrated (BNPL 14d, BNPL 30d, Pay in 3, Direct Debit, Debt Collection, ...)
- **phase**: Current onboarding phase (see phases below)
- **go_live**: Target go-live date

## Onboarding Phases

1. **Discovery** — Initial contact, requirements clarification, scope definition
2. **Contract** — Contract negotiation, legal review, compliance checks
3. **Tech Integration** — API/plugin integration, technical implementation by merchant
4. **UAT** — User Acceptance Testing, test transactions, sign-off
5. **Go-Live** — Production launch, monitoring
6. **Hypercare** — Post-go-live support period (typically 2–4 weeks)

## Standard Internal Contacts (Riverty)

- Onboarding Project Manager
- Technical Implementation Manager
- Account Manager
- Legal / Compliance
- Risk Team
- Product Team

## Standard External Contacts (Merchant)

- Merchant Project Manager
- Merchant Technical Contact
- Merchant Legal / Commercial Contact

## Typical Documents

- `emails/` — Email correspondence with merchant
- `meeting-notes/` — Kickoff, alignment, UAT review meetings
- `misc/` — Contracts, technical specs, test protocols, checklists

## Typical Communication Needs

- Kickoff email to merchant
- Weekly status update to internal stakeholders
- Technical alignment meeting summary
- UAT results summary
- Go-live announcement
- Escalation communication to management
- Hypercare status update
