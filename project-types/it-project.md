# IT Project

## Key Fields

- **phase**: Current project phase
- **go_live**: Target go-live / deployment date
- **budget**: Project budget

## Standard Phases

1. **Discovery** — Requirements, stakeholder alignment
2. **Design** — Architecture, technical design
3. **Development** — Implementation
4. **Testing** — QA, UAT
5. **Deployment** — Go-live
6. **Support** — Hypercare / ongoing support

## Standard Internal Contacts

- Project Manager
- Tech Lead / Architect
- Product Owner
- Developer(s)
- QA / Testing
- Operations / DevOps
- Business Stakeholder

## Standard External Contacts

- Vendor / Supplier
- External Consultant
- Integration Partner

## Typical Documents

- `emails/` — Stakeholder correspondence
- `meeting-notes/` — Sprint reviews, planning sessions, escalations
- `misc/` — Requirements docs, architecture diagrams, test plans

## Typical Communication Needs

- Project kickoff announcement
- Sprint / milestone status update
- Stakeholder briefing
- Incident / escalation communication
- Go-live announcement
- Post-mortem / lessons learned
